## Commands

```bash
npm run dev          # Start dev server (also pulls i18nexus translations)
npm run build        # Production build
npm run lint         # ESLint
npm test             # Run tests once (Vitest)
npm run test:watch   # Run tests in watch mode
npm run deploy       # Build + deploy to Cloudflare Workers
npm run preview      # Preview on Cloudflare locally
npm run cf-typegen   # Regenerate Cloudflare Worker types
npm run vector:upsert  # Seed Cloudflare Vectorize with embeddings
npm run vector:query   # Test vector search
```

Run a single test file: `npx vitest run tests/api/askai.test.ts`

## Architecture

**Stack:** Next.js 16 App Router, React 19, TypeScript (strict), Tailwind CSS 4, Cloudflare Workers (via @opennextjs/cloudflare), Vitest.

**Theme:** VS Code-inspired IDE aesthetic — left sidebar navigation, center MDX content panel, bottom status bar.

### Routing & i18n

All routes live under `app/[locale]/`. Locales are `en` (default) and `de`, managed by **next-intl** with `localePrefix: 'always'`. Page content is split into locale-specific MDX files (`page-en.mdx`, `page-de.mdx`) that are dynamically imported based on the active locale. Use the i18n-aware wrappers from `i18n/navigation.ts` (`Link`, `useRouter`, etc.) — never the plain Next.js ones.

Translation strings live in `messages/en.json` and `messages/de.json` and are synced from i18nexus on `npm run dev`.

### API Routes (`app/api/`)

- **`/api/askai`** — RAG endpoint: embeds the user prompt with `Xenova/all-MiniLM-L6-v2` (via HuggingFace Transformers), queries **Cloudflare Vectorize** for top-3 matches, then sends prompt + context to **Groq** (Llama 3.1 8B via LangChain) and returns `{ answer, embedding }`.
- **`/api/emails`** — Contact form: validates with a Zod schema in `lib/`, sends via **Resend**.
- **`/api/hello`** — Demo/test endpoint.

### AI / Vector DB

The RAG pipeline stores portfolio content embeddings in Cloudflare Vectorize. `scripts/upsert-vector-db.ts` seeds the database. The `askai` route handles embedding + retrieval + generation at request time. Cloudflare bindings (`VECTORIZE`, etc.) are defined in `worker-configuration.d.ts` (regenerated by `npm run cf-typegen`).

### Validation

Zod schemas for the contact form (field limits, error messages) live in `lib/`. The same schema is used on both the server (API route) and client (contact form component).

### MDX Pages

Project pages, legal pages, and the home page are MDX files under `app/[locale]/`. The `@next/mdx` plugin handles compilation. MDX is rendered with Tailwind Typography styles.

## Deployment

Deployed to **Cloudflare Workers** via GitHub Actions (`.github/workflows/deploy.yml`) on every push to `main`. Required secrets: `CLOUDFLARE_API_TOKEN`, `CLOUDFLARE_ACCOUNT_ID`, `GROQ_API_KEY`, `RE_SEND_API_KEY`.

Local env vars go in `.env.local`: `NEXT_URL`, `I18NEXUS_API_KEY`, `RE_SEND_API_KEY`, `GROQ_API_KEY`.

## Testing

Tests are in `tests/api/` and run in a Node environment (not jsdom). AI-related tests have a 120 s timeout due to LLM calls. Keep tests integration-style; the test suite hits real LLM/vector endpoints in CI.
